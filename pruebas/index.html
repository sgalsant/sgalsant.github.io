Pruebas de software

Las pruebas no son opcionales, se deben realizar.

Las pruebas se automatizan en la medida de lo posible.

Tipos de pruebas funcionales:

  • Pruebas unitarias.
  • Pruebas de integración.
  • Pruebas de aceptación.
  • Pruebas de integración.
  • Pruebas de regresión.

Tipos de pruebas no funcionales:

  • Pruebas de carga.
  • Pruebas de estrés.
  • Pruebas de escalabilidad.
  • Pruebas de portabilidad.
  • Pruebas de seguridad
  • Pruebas de usabilidad

Pruebas unitarias

Se realiza sobre una unidad de desarrollo:función o una clase

Pruebas integración

En la prueba interviene diferentes funciones y/o clases.

Pruebas regresión

Se vuelve a realizar las pruebas tras la modificación del software

Pruebas aceptación

Pruebas sobre todo el software. Prueba la funcionalidad final del software.

Pruebas integración continua

Los test se ejecutan de forma automática y periódica.

TDD Test Driven Developement

En castellano: Desarrollo guiado por pruebas

TDD Test Driven Developement

Primero escribimos los test que expresan los requerimientos a cumplir, luego desarrollamos para cumplir con dichos requerimientos.

TDD Test Driven Developement

Si se aprecia un error, primero se crea un test que identifica el error y luego se modifica el código para que pase el test

Aplicando TDD conseguimos un código más robusto, más seguro, más mantenible y una mayor rapidez en el desarrollo

Instalar Mocha + Chai

Ejecutamos en la carpeta de nuestro proyecto:

npm i -D mocha
npm i -D chai

Creamos en nuestro proyecto una carpeta llamada "test"

En esta carpeta vamos a poner todos los ficheros con los tests: *.test.js

Además, en la carpeta raíz situamos la página html para ejecutar los tests: test.html

La plantilla de esta página se encuenta en el directorio "node_modules/mocha"
